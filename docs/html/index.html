WhatsApp User Experience

Users interact with the AI assistant directly through WhatsApp. Here's how it works:

Natural Language Commands
Users simply send messages like "Remind me to call mom at 5pm" or "Schedule meeting tomorrow at 2pm" and the AI understands and executes.
Proactive Reminders
The AI sends WhatsApp notifications at scheduled times. Supports one-time, daily, weekly, and monthly recurring reminders.
Calendar Integration
Users can say "Schedule team meeting Friday at 3pm" and the AI creates calendar events. Syncs with Google Calendar and Outlook.
Document Q&A
Upload documents via dashboard, then ask questions in WhatsApp like "What does the contract say about payment terms?"
Smart Suggestions
AI analyzes patterns and suggests reminders proactively. "I noticed you call your doctor every Monday. Want me to set a weekly reminder?"
Shared Reminders
Share reminders with family or team members. "Remind John about the presentation" sends notification to both users.

Service Mesh (Istio)

  • mTLS in STRICT mode for all service-to-service communication
  • Circuit breakers with configurable thresholds
  • Rate limiting and quota management
  • JWT validation for API authentication
  • Automatic retries with exponential backoff
  • Traffic management for canary deployments

Monitoring & Observability

Comprehensive monitoring with Prometheus, Grafana, and custom dashboards for production-grade observability.

Monitoring Stack

Prometheus
Metrics collection with 100+ custom metrics for services and infrastructure.
Grafana
Dashboards for system health, service metrics, and business KPIs.
Loki
Log aggregation with label-based filtering and Grafana integration.
Tempo
Distributed tracing with OpenTelemetry for request flow analysis.

Key Dashboards

  • System Overview - Cluster health, node metrics, pod status
  • Service Metrics - Request rate, latency, error rates per service
  • Agent Performance - LangGraph execution times, token usage
  • RAG Quality - Search relevance, retrieval accuracy
  • Business KPIs - Active users, message volume, AI response times
  • Database Performance - Query latency, connection pools, cache hit rates

Comprehensive setup script that:

  • Generates Prisma client
  • Creates environment configuration files
  • Builds all packages and services
  • Sets up Docker containers

./scripts/dev.sh

Starts all services in development mode using Turbo with parallel execution.

Deployment Script

./scripts/deploy.sh [environment]

Deploy to different environments:

  • dev - Local development deployment
  • staging - Staging environment with K8s
  • production - Production deployment (default)

Health Check Script

./scripts/health-check.sh

Checks health status of all services:

  • API Gateway (health, ready)
  • Dashboard
  • PostgreSQL, Redis, MongoDB
  • AI Services (Agent, RAG, WhatsApp API)

Migration Script

./scripts/migrate.sh [command]

Database migration management:

  • status - Check migration status
  • up - Run pending migrations
  • down - Rollback last migration
  • deploy - Deploy migrations (production)
